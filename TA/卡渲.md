# HighLightMatCap
## 为什么「法线・相机 Right」「法线・相机 Up」能当成 UV？

你现在这一步，本质上是把**模型表面的法线方向，转换成了一张 2D 球形贴图的坐标**：

1.  相机的 `Right` 和 `Up` 向量，构成了相机视角的局部坐标系（水平 + 垂直轴）。
2.  用世界法线和这两个轴做点积，就是把法线投影到这两个轴上，得到 `[-1,1]` 范围的两个标量值。
3.  把这两个值拼成 `float2`，再映射到 `[0,1]`，就成了 UV。

这个 UV 的特殊之处在于：

-   模型上**正对着相机的面**，法线在 Right 和 Up 上的投影都接近 0，UV 接近 (0.5, 0.5)，会采样贴图中心；
-   模型上**朝相机侧面的面**，法线在 Right/Up 上的投影接近 ±1，UV 会跑到贴图边缘；
-   你旋转相机 / 模型时，法线和相机轴的相对方向变了，UV 也会跟着变，采样的贴图位置就变了。

所以，只要你在贴图的中心画高光、边缘画暗部，就能模拟出一个永远跟着相机视角走的高光效果。

## 二、为什么这能 “模拟高光”？它和真实高光的本质区别

### ✨ 真实高光的原理（PBR/Phong）

真实高光，是计算**光线反射方向和相机视角的夹角**：

1.  先算出入射光的反射向量；
2.  再算反射向量和相机视角方向的点积；
3.  点积越大，高光越亮，反之越暗。
4.  高光的形状、强度，完全由光照方向、法线、相机位置三者决定，每帧都要实时计算。

### 🎨 MatCap 高光的原理

MatCap 高光，是把 **“不同视角方向对应的高光亮度 / 颜色”，提前画在一张贴图里 **：

1.  用法线和相机轴的投影，算出当前视角对应的贴图 UV；
2.  直接采样贴图里的颜色，作为高光结果；
3.  不用管光照方向，不用算反射向量，一采样就出结果。

它本质上是一个 **“视角方向→高光颜色” 的查表法 **，用一张贴图代替了复杂的光照计算。

## 三、教程为什么推荐用 MatCap，而不是直接算真实高光？
**风格化效果**
MatCap 高光：可以直接在贴图里画出卡通、手绘、金属、皮肤等任意质感，自由度极高
真实高光：只能基于物理公式计算，很难做出夸张 / 艺术化的高光

**性能开销**
MatCap 高光：只需要一次贴图采样，开销极低
真实高光：需要计算反射向量、点积、幂运算，多光源下开销更高
**视角一致性**
MatCap 高光：高光永远跟着相机走，不管怎么转视角，效果都稳定
真实高光：高光位置由光照方向决定，相机转了高光不会跟着动，容易出违和感

**无光照环境**
MatCap 高光：不用实时光照，也能做出 “被照亮” 的效果，适合烘焙光照 / 无光照场景
真实高光：必须依赖光源，没有光就没有高光

**快速迭代**
MatCap 高光：换一张贴图就能换一套高光质感，美术可以直接在 PS 里画，不用调参数
真实高光：要反复调整高光强度、粗糙度、菲涅尔等一堆参数，迭代慢

## 四、MatCap 的局限性：它不是万能的

MatCap 高光也有明显的缺点，你要心里有数：

1.  **不支持多光源**：它的高光只跟着相机走，不能响应场景里的其他光源，比如你加个点光源，MatCap 高光不会跟着变；
2.  **无法区分环境光和直射光**：它的效果是 “固定的视角高光”，不能像真实高光那样区分漫反射、镜面反射、环境反射；
3.  **视角限制**：极端视角下，相机坐标系构建不稳定，高光会出现扭曲 / 跳变。

----------

## 五、给你一个通俗的比喻

你可以把 MatCap 贴图想象成一个 **“球形的高光模板”**：

-   模板的中心是高光，边缘是暗部；
-   模型上的每个面，都根据自己朝向相机的角度，在模板上找对应的位置取颜色；
-   不管你怎么转相机，每个面都能在模板上找到正确的高光位置，看起来就像高光永远跟着视角走。

而真实高光，相当于每次都根据光照和视角，现场 “画” 出高光，虽然真实，但慢、不灵活，也很难做出夸张的风格化效果。
