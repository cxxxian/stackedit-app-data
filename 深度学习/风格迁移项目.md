## 基础概念速查

### 通道（Channel）≠ 分辨率

```
分辨率 = 每层特征图的空间大小（如 64×64）
通  道 = 叠了多少层特征图

RGB 图像 [3, 64, 64]：
  通道1（R）：64×64 记录红色强度
  通道2（G）：64×64 记录绿色强度
  通道3（B）：64×64 记录蓝色强度

经过 Conv2d(3→64) 后：
  64个通道，每个通道记录一种特征（边缘方向、纹理等）
```

### 卷积层（Conv2d）——提取特征，图像缩小

```python
nn.Conv2d(in_channels, out_channels, kernel_size=4, stride=2, padding=1)
# stride=2 → 图像尺寸减半，通道数变为 out_channels
```

- 用卷积核在图像上滑动扫描，每个位置聚合一片区域 → 输出一个数
- **多个输入像素 → 一个输出像素**（图缩小）

### 转置卷积（ConvTranspose2d）——图像放大

```python
nn.ConvTranspose2d(in_channels, out_channels, kernel_size=4, stride=2, padding=1)
# stride=2 → 图像尺寸翻倍
```

- 把每个输入数"扩散"到输出的一片区域
- **一个输入像素 → 多个输出像素**（图放大）
- 卷积核同样是可学习的参数

### 激活函数

| 函数 | 规则 | 用在哪 |
|------|------|--------|
| **LeakyReLU(0.2)** | 正数直通，负数×0.2 | 编码器（下采样）中间层 |
| **ReLU** | 正数直通，负数→0 | 解码器（上采样）中间层 |
| **Tanh** | 输出压缩到 [-1, 1] | 最后输出层（和输入归一化范围匹配） |

> 没有激活函数 → 不管叠多少层都等于一层，无法学复杂规律

### 归一化（InstanceNorm2d）

把每层输出的数值稳定到合理范围，让训练更稳定、更快。  
不改变特征内容，只调整数值范围。

### 权重初始化

```python
torch.nn.init.normal_(m.weight.data, 0.0, 0.02)  # 均值0，标准差0.02
```

- **不能全部初始化为同一个数**：会导致所有神经元完全相同（对称性问题），训练后永远是复制品，等于只有1个神经元
- 随机初始化打破对称，让每个神经元学不同特征

### 训练的四步循环

```
① 前向传播：数据流过网络，得到输出
② 计算 Loss：输出和目标之间的差距
③ 反向传播 loss.backward()：PyTorch 自动计算每个参数的梯度
④ 参数更新 optimizer.step()：按梯度方向调整所有参数（上下采样的卷积核都会更新）
```

### L1 Loss vs L2 Loss

| | L1 Loss | L2 Loss |
|--|---------|---------|
| 公式 | `|预测 - 真实|` | `(预测 - 真实)²` |
| 对大误差 | 线性惩罚 | 平方惩罚（更严厉） |
| 图像效果 | 轻微模糊 | 更模糊 |
| 使用场景 | 图像生成首选 | 不适合图像生成 |

> L1/L2 都会让图像偏模糊，因为网络学会"取像素平均值"来最小化损失

## Autoencoder 自编码器

### 核心思路

> 压缩再还原。用编码器把图像压缩成"精华摘要"（latent），再用解码器从摘要重建图像。

### 网络结构

```
输入图 [1, 3, 64, 64]
  ↓
【Encoder 编码器】
  down1: Conv(3→64)    → [1, 64,  32, 32]   浅层特征（边缘等）
  down2: Conv(64→128)  → [1, 128, 16, 16]   中层特征
  down3: Conv(128→256) → [1, 256,  8,  8]   高层特征
  down4: Conv(256→512) → [1, 512,  4,  4]   latent（字的精华摘要）
  ↓
【Decoder 解码器】
  up1: ConvT(512→256) → [1, 256,  8,  8]
  up2: ConvT(256→128) → [1, 128, 16, 16]
  up3: ConvT(128→64)  → [1, 64,  32, 32]
  final: ConvT+Conv+Tanh → [1, 3, 64, 64]
  ↓
重建图（和输入一样大）
```

### 损失函数

```python
loss = L1Loss(reconstructed, real_imgs)
# 逐像素求差的绝对值取平均，越小越好
```

### 训练方式

```python
real_imgs = batch['A']          # 输入 = 目标 = 同一张楷体图
reconstructed = autoencoder(real_imgs)
loss = L1Loss(reconstructed, real_imgs)
loss.backward()
optimizer.step()
```

- **1个网络，1个优化器，1个损失**
- 输入和目标是同一张图（A→A 重建，不是风格迁移）

### 优缺点

- ✅ 结构简单，训练稳定
- ❌ 只有 L1 Loss → 输出偏模糊
- ❌ 不是风格迁移（A→A），只能重建

## GAN 生成对抗网络

### 核心思路

> 两个网络互相博弈：生成器（画家）不断学着骗过判别器（鉴定师），判别器不断学着识别真假，最终生成器生成的图连判别器都分不清。

### 两个网络

#### 生成器（Generator）

结构和 Autoencoder 的编解码器几乎一样，但：
- **输入是 B（黑体），目标是 A（楷体）** → 真正的风格迁移
- 训练时同时受两个损失监督

```
输入 real_A（楷体） → 生成器 → fake_B（生成的黑体）
```

#### 判别器（Discriminator / PatchGAN）

```python
def forward(self, img_A, img_B):
    img_input = torch.cat((img_A, img_B), 1)  # 两张图拼在一起（6通道）
    return self.model(img_input)               # 输出 4×4 真假分数矩阵
```

- 输入：条件图 + 目标/生成图（拼在一起，6通道）
- 输出：**4×4 矩阵**（PatchGAN，每个数评判一个局部区域的真假）

**为什么用 PatchGAN（局部判断）？**
- 整图判断：只看全局，生成器可以用"大体像"来蒙混
- PatchGAN：16个位置分别打分，每个笔画细节都要过关

**PatchGAN 怎么算出 4×4 矩阵的？**
```
输入 [1, 6, 64, 64]
  ↓ Conv(6→64, stride=2)    → [1, 64,  32, 32]
  ↓ Conv(64→128, stride=2)  → [1, 128, 16, 16]
  ↓ Conv(128→256, stride=2) → [1, 256,  8,  8]
  ↓ Conv(256→512, stride=2) → [1, 512,  4,  4]
  ↓ Conv(512→1)             → [1,   1,  4,  4]  ← 4×4 真假矩阵
```
不是人工切块，是卷积自然下采样的结果，每个输出数天然对应原图的一个局部区域（感受野）。

### 损失函数

```python
criterion_GAN       = MSELoss()   # 对抗损失（判断真假）
criterion_pixelwise = L1Loss()    # 像素损失（像不像）
lambda_pixel        = 20          # L1 的权重
```

**两个损失的职责：**

| 损失 | 问的问题 | 作用 |
|------|----------|------|
| **像素损失 L1** | "生成的字和目标字像素一样吗？" | 保证结构正确，但容易模糊 |
| **对抗损失 GAN** | "生成的字看起来像真实字体吗？" | 保证视觉清晰，笔画锋利 |
| **合并** | `loss_G = loss_GAN + 20×loss_pixel` | 又准确又清晰 |

### 训练流程（每个 batch）

```
读入数据：real_A（楷体输入），real_B（黑体目标）

━━━ 阶段1：训练生成器 G ━━━
① fake_B = generator(real_A)               生成假图
② pred_fake = discriminator(fake_B, real_A) 判别器打分
③ loss_GAN = MSE(pred_fake, 全1矩阵)        希望判别器给高分（认为是真的）
④ loss_pixel = L1(fake_B, real_B)           希望像素接近真实目标
⑤ loss_G = loss_GAN + 20×loss_pixel
⑥ loss_G.backward() + optimizer_G.step()   更新生成器

━━━ 阶段2：训练判别器 D ━━━
① pred_real = discriminator(real_B, real_A) 真实对打分
② loss_real = MSE(pred_real, 全1矩阵)       真实对→高分
③ pred_fake = discriminator(fake_B.detach(), real_A)  假图打分
   # .detach() 切断梯度，防止影响生成器
④ loss_fake = MSE(pred_fake, 全0矩阵)       假图→低分
⑤ loss_D = 0.5×(loss_real + loss_fake)
⑥ loss_D.backward() + optimizer_D.step()   更新判别器
```

### 优缺点

- ✅ 真正的风格迁移（B→A）
- ✅ 比 Autoencoder 更清晰（GAN Loss 对抗模糊）
- ❌ 无跳跃连接，编码过程中细节丢失，笔画不够锋利

## Pix2Pix

### 核心思路

> 在 GAN 基础上加入**跳跃连接（Skip Connection）**：编码时保存每层的特征快照，解码时把对应层的快照拼进来，让细节永远不丢失。

### 和 GAN 唯一的区别：生成器加了跳跃连接

判别器结构完全相同（PatchGAN）。

### 跳跃连接实现

```python
# UNetUp 解码块的 forward
def forward(self, x, skip_input):
    x = self.model(x)                      # ① 转置卷积上采样（图放大，通道减少）
    x = torch.cat((x, skip_input), 1)     # ② 把编码器的快照拼进来（通道加倍）
    return x
```

`torch.cat(..., 1)`：在通道维度拼接，空间大小不变，通道数相加。

### 完整数据流（带尺寸）

```
输入 [1, 3, 64, 64]
  ↓ down1
d1 [1, 64, 32, 32]  ←─────────────────────────────────┐ 快照存下来
  ↓ down2                                               │
d2 [1, 128, 16, 16] ←────────────────────────┐         │
  ↓ down3                                     │         │
d3 [1, 256, 8, 8]   ←───────────────┐        │         │
  ↓ down4                            │        │         │
d4 [1, 512, 4, 4]                   │        │         │
  ↓ up1(d4, d3)                     │        │         │
  ConvT(512→256): [1,256,8,8]       │        │         │
  cat(256+256=512):                 ┘        │         │
u1 [1, 512, 8, 8]                            │         │
  ↓ up2(u1, d2)                              │         │
  ConvT(512→128): [1,128,16,16]              │         │
  cat(128+128=256):                          ┘         │
u2 [1, 256, 16, 16]                                    │
  ↓ up3(u2, d1)                                        │
  ConvT(256→64): [1,64,32,32]                          │
  cat(64+64=128):                                      ┘
u3 [1, 128, 32, 32]
  ↓ final: Upsample + Conv(128→3) + Tanh
输出 [1, 3, 64, 64]
```

**为什么 up 层的 in_size 不是上一层的 out_size？**

```python
self.up1 = UNetUp(512, 256)  # in=512（d4本身）
self.up2 = UNetUp(512, 128)  # in=512（不是256！因为cat后：256+256=512）
self.up3 = UNetUp(256, 64)   # in=256（不是128！因为cat后：128+128=256）
```

cat 之后通道数翻倍，所以下一层的输入通道数要按 cat 后的来。

### 跳跃连接的本质

```
d1/d2/d3 是楷体的位置/细节信息（"笔画在哪里"）
解码器特征是正在生成的黑体风格信息（"画什么风格"）

拼在一起后，下一层卷积学会融合：
  "在楷体快照指定的精确位置，画出黑体风格的笔画"

类比：临摹时旁边放着原图对照位置，但风格完全是自己画的
```

### 优缺点

- ✅ 风格迁移效果最好
- ✅ 跳跃连接保留细节，笔画锋利
- ✅ 位置准确（参考了楷体的空间信息）

## 三种方法对比总结

### 架构对比

| | Autoencoder | GAN | Pix2Pix |
|--|-------------|-----|---------|
| **网络数量** | 1个 | 2个（G+D） | 2个（G+D） |
| **输入→目标** | A→A（重建） | B→A（迁移） | B→A（迁移） |
| **跳跃连接** | ❌ | ❌ | ✅ |
| **判别器** | ❌ | ✅ PatchGAN | ✅ PatchGAN |
| **损失函数** | L1 | L1 + GAN | L1 + GAN |
| **优化器数量** | 1个 | 2个 | 2个 |

### 效果对比

```
Autoencoder  →  结构大致对，但严重模糊，无法做风格迁移
GAN          →  可以做风格迁移，比 Autoencoder 更清晰，但细节有丢失
Pix2Pix      →  风格迁移最好，笔画清晰，细节保留最完整  ← 最优
```

### 演进逻辑

```
Autoencoder（会重建图像）
    │
    │ 问题：只有 L1 Loss 导致模糊，且不能风格迁移
    ↓
GAN（加入判别器 + 改为 B→A 训练）
    │
    │ 问题：无跳跃连接，笔画细节在编码过程中丢失
    ↓
Pix2Pix（加入跳跃连接）
    │
    ✓ 又准确又清晰，是本项目效果最好的方案
```

### 为什么模糊问题能被 GAN 解决？

```
L1 Loss 问："你的像素值和答案一样吗？"
  → 网络学会取所有可能输出的平均 → 模糊

GAN Loss 问："你的输出看起来像真实字体吗？"
  → 判别器见过大量真实字体，知道真实字体是锐利的
  → 模糊的输出会被判别器识破 → 生成器被惩罚
  → 生成器被迫生成清晰的笔画
```
